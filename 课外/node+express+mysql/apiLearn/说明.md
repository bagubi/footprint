### 基础项目结构
apiLearn/
├── app.js                           # 主应用入口文件，配置Express应用和中间件
├── package.json                     # 项目配置文件，包含依赖、脚本和元数据
├── 说明.md                          # 项目说明文档，包含学习指南和结构说明

├── models/                          ###################################### 数据模型层目录，处理数据访问逻辑

│   └── youyuController.js           # youyu控制器，协调路由和模型之间的交互
├── views/                           ###################################### 视图层目录，存放模板文（Jade模板）

└── public/                          ###################################### 静态资源目录，存放前端静态件
    ├── stylesheets/                 # CSS样式文件

    └── images/                      # 图片资源

# Node.js + Express + MySQL 应用程序的分层架构
## 典型的三层架构
1. Model（模型）层
职责：负责数据访问和数据库操作
示例：youyuModel.js 文件
功能：执行 SQL 查询、数据验证、数据映射
2. Controller（控制器）层
职责：处理业务逻辑和请求响应
示例：youyuController.js 文件
功能：接收请求、调用模型层、返回响应
3. Route（路由）层
职责：定义 URL 映射和请求分发
功能：将不同 URL 映射到对应的控制器方法
## 更详细的五层架构（扩展版）
1. Router（路由）层
定义 API 端点和 HTTP 方法
将请求转发给对应的控制器
2. Controller（控制器）层
业务逻辑处理
协调模型层和视图层
3. Model（模型）层
数据访问和数据库操作
业务数据实体
4. Service（服务）层（可选）
复杂业务逻辑封装
跨多个模型的操作
5. Utility（工具）层
ResponseHandler - 统一响应格式
dbconfig - 数据库连接配置
日志、验证等通用功能
